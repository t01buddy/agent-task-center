# Data Models

SQLite schema for Agent Task Center. All tables use `TEXT` primary keys (UUIDs generated by the service), except where noted.

---

## workspaces

```sql
CREATE TABLE workspaces (
    id         TEXT PRIMARY KEY,           -- UUID
    name       TEXT NOT NULL UNIQUE,       -- user-defined label, e.g. "personal"
    created_at TEXT NOT NULL               -- ISO 8601 UTC
);
```

---

## agents

```sql
CREATE TABLE agents (
    id                  TEXT PRIMARY KEY,  -- caller-provided or UUID
    name                TEXT NOT NULL,
    runtime             TEXT NOT NULL,     -- e.g. "claude-code", "codex", "custom"
    runtime_version     TEXT,
    domain              TEXT NOT NULL,     -- user-defined group, e.g. "review", "codex", "any"
    workspace_id        TEXT REFERENCES workspaces(id),
    capabilities        TEXT,              -- JSON array of strings
    last_heartbeat_at   TEXT,              -- ISO 8601 UTC
    status              TEXT NOT NULL DEFAULT 'active',
                                           -- active | stale | offline
    registered_at       TEXT NOT NULL      -- ISO 8601 UTC
);

CREATE INDEX idx_agents_domain      ON agents(domain);
CREATE INDEX idx_agents_workspace   ON agents(workspace_id);
CREATE INDEX idx_agents_status      ON agents(status);
```

---

## task_types

```sql
CREATE TABLE task_types (
    id                           TEXT PRIMARY KEY,  -- UUID
    name                         TEXT NOT NULL UNIQUE,
    default_visibility_timeout_s INTEGER NOT NULL DEFAULT 300,
    max_attempts                 INTEGER NOT NULL DEFAULT 3,
    retry_backoff_s              INTEGER NOT NULL DEFAULT 60,
    hard_deadline_s              INTEGER,           -- NULL = no hard deadline
    stale_heartbeat_threshold_s  INTEGER NOT NULL DEFAULT 60,
    created_at                   TEXT NOT NULL
);
```

---

## tasks

```sql
CREATE TABLE tasks (
    id                TEXT PRIMARY KEY,             -- UUID
    workspace_id      TEXT REFERENCES workspaces(id),
    domain            TEXT,                         -- optional domain filter
    task_type_id      TEXT REFERENCES task_types(id),
    title             TEXT NOT NULL,
    priority          INTEGER NOT NULL DEFAULT 0,   -- higher = more urgent
    context           TEXT,                         -- JSON object
    status            TEXT NOT NULL DEFAULT 'queued',
                      -- queued | leased | running | blocked |
                      -- completed | failed | timed_out | cancelled
    assigned_agent_id TEXT REFERENCES agents(id),
    lease_expires_at  TEXT,                         -- ISO 8601 UTC; NULL when not leased
    retry_after       TEXT,                         -- ISO 8601 UTC; NULL unless backing off
    attempt_count     INTEGER NOT NULL DEFAULT 0,
    created_at        TEXT NOT NULL,
    updated_at        TEXT NOT NULL
);

CREATE INDEX idx_tasks_status       ON tasks(status);
CREATE INDEX idx_tasks_workspace    ON tasks(workspace_id);
CREATE INDEX idx_tasks_domain       ON tasks(domain);
CREATE INDEX idx_tasks_task_type    ON tasks(task_type_id);
CREATE INDEX idx_tasks_agent        ON tasks(assigned_agent_id);
CREATE INDEX idx_tasks_priority     ON tasks(priority DESC, created_at ASC);
CREATE INDEX idx_tasks_lease_expiry ON tasks(lease_expires_at) WHERE status = 'leased';
```

**Status state machine:**

```
queued → leased → running → completed
                           → failed
       ↗ (retry)           → timed_out (attempts exhausted)
leased → timed_out (lease expired, attempts exhausted)
queued → cancelled
blocked → queued (manual unblock, v2)
blocked → cancelled
```

## task_attempts

One row per lease/attempt. Records the lifecycle of a single agent's claim on a task.

```sql
CREATE TABLE task_attempts (
    id            TEXT PRIMARY KEY,   -- UUID
    task_id       TEXT NOT NULL REFERENCES tasks(id),
    agent_id      TEXT REFERENCES agents(id),
    fencing_token INTEGER NOT NULL,   -- monotonically increasing per task
    started_at    TEXT NOT NULL,      -- ISO 8601 UTC
    expires_at    TEXT NOT NULL,      -- ISO 8601 UTC (initial; heartbeat extends task row)
    ended_at      TEXT,               -- NULL while active
    result_code   TEXT                -- completed | failed | timed_out | superseded
);

CREATE INDEX idx_attempts_task ON task_attempts(task_id);
```

`fencing_token` is `MAX(fencing_token) + 1` for the task at lease time (starts at 1).

## task_events

Append-only audit log. Never updated or deleted.

```sql
CREATE TABLE task_events (
    id         TEXT PRIMARY KEY,      -- UUID
    task_id    TEXT NOT NULL REFERENCES tasks(id),
    attempt_id TEXT REFERENCES task_attempts(id),
    agent_id   TEXT REFERENCES agents(id),
    event_type TEXT NOT NULL,
               -- created | leased | heartbeat | progress |
               -- completed | failed | timed_out | retried |
               -- cancelled | custom
    payload    TEXT,                  -- JSON object; event-type-specific data
    created_at TEXT NOT NULL          -- ISO 8601 UTC
);

CREATE INDEX idx_events_task ON task_events(task_id, created_at);
```

---

## task_logs

Structured log lines attached to a task/attempt/agent.

```sql
CREATE TABLE task_logs (
    id         TEXT PRIMARY KEY,      -- UUID
    task_id    TEXT NOT NULL REFERENCES tasks(id),
    attempt_id TEXT REFERENCES task_attempts(id),
    agent_id   TEXT REFERENCES agents(id),
    level      TEXT NOT NULL,         -- debug | info | warn | error
    message    TEXT NOT NULL,
    created_at TEXT NOT NULL          -- ISO 8601 UTC
);

CREATE INDEX idx_logs_task    ON task_logs(task_id, created_at);
CREATE INDEX idx_logs_agent   ON task_logs(agent_id, created_at);
CREATE INDEX idx_logs_level   ON task_logs(level);
```

---

## Migration Strategy

- Migrations are numbered SQL files embedded in the binary (`migrations/0001_init.sql`, etc.).
- Applied in order on startup; idempotent (tracked in a `schema_migrations` table).
- No external migration tool required.
